# course/ — 講義制課程記錄（2026-07-20 起）

W1 實授後課程轉向「講義制」，Slidev deck 生產線暫停——背景與完整決策見
[DECISIONS.md「課程大改」](../DECISIONS.md)。本資料夾是新產線在本 repo 的唯一落點。

## 產線分工

- **內容（Claude Cowork）**：當週講義的設計、MD 撰寫、MD→HTML 轉換、定稿全在 Cowork
  project 完成；講義本體（MD/HTML）**不進本 repo**。
- **發放**：講義走 LINE 等行政管道直接發檔；儀表板每週 data.js 磚資料與 skills 照舊放
  外部模板庫 `github-actions-cicd-course` 的 `W{N}/`（學員照 W1 學過的方式取檔）。
- **本 repo（轉譯層）**：收到當週定稿講義 → 生成 `course/week-N.md`。目的＝把新內容轉譯成
  舊架構語彙，讓 W6 後整合回本專案做對外公開版時有現成對照；**不承擔課堂實錄**（實際教學
  進度落差由講師口頭掌握）。

## week-N.md 格式（一週一檔，curriculum＋kit 同檔）

- **Curriculum 節（大綱層）**：P1–P7 堂次分段（網格保留、每堂密度低），每段＝段落標題＋
  一句內容摘要。不用【NN】編號、不用頁型標記、不標頁數——舊 docs 的頁級體系已整套凍結，
  頁級切分留到真的要做 deck 時再下。
- **Kit 節（轉譯重點——儀表板是唯一有每週死線的消費者）**：
  - **知識點清單**：`- >PN — 堂標題` 分組、底下 `- [ ] 關鍵字`，與儀表板磚 1:1
    （磚號＝堂號）；顆粒度沿用舊慣例——指令列舉型每指令一勾、概念型整塊一勾、
    旗標／變體折進主指令。
  - **skills／模板發放**：當週從模板庫 `W{N}/` 發什麼、學員怎麼取。
- 一律以收到的講義**照實轉譯**；週間編排（如 W1 缺口的補課位置）由 Cowork 端逐週決定，
  這裡不預判。

## 講義本身的固定約定（Cowork 端遵循，此處備查）

- 文件風**單檔 HTML、一份兩用**：上課投影開同一份捲動講；原檔發學員留存，可自行列印 PDF。
- **第一章固定「環境重建」**：約 1/3 學員用教室設備、關機還原，每週開場照這章重裝。
- spec.csv 鬆綁為參照（不再維護差異表）；內容密度以「逾 2/3 非本科新手＋年長學員」的
  實際吸收為準，每堂預留巡場排障時間。
